Goal
- Implement the paper as a new logical-separation method alongside the existing `orb` and `hybrid` options.
- Keep the existing methods intact and add a third method that follows the paper's stable-keypoint template-construction pipeline.
- Target use case is better recall on scanner-produced pages rendered from PDFs.
- Chosen approach: use SURF exactly for the new paper-based method.

Paper note
- No, the paper does not use ORB in its reported experiments; it uses SURF.
- The paper's stability idea is detector-agnostic in principle, but an exact implementation is SURF-based unless we intentionally adapt it.
- The current repo depends on `opencv-python`, not `opencv-contrib-python`, so SURF support is not present yet.
- Decision: replace the backend OpenCV dependency with `opencv-contrib-python` so the new method can use SURF.

Why this is relevant
- The repo already does template-based page-start detection with feature matching and homography in `backend/app/api/extraction.py`.
- Adding the paper as a separate method fits the existing method selector and lets you compare current `orb` and `hybrid` behavior against the paper-based pipeline on your PDFs.

Implementation plan
1. Add a new paper-specific separation method end to end.
   - Extend backend method parsing and the logical-separation response flow to accept a third method id.
   - Extend the frontend separation method selector and types so the new method appears in the UI.
   - Keep `orb` and `hybrid` unchanged.

2. Add the paper's varying-field hiding phase before feature selection.
   - Extend template data so the paper method can exclude pre-marked document regions whose content varies between exemplars.
   - Chosen approach: store separate ignore/mask regions for the paper method instead of reusing extraction zones.
   - Build the paper-method template only from keypoints detected outside those masked regions, matching the paper's "outside the pre-marked document zones" step.
   - Ensure the stability computation and retained descriptor set ignore masked regions entirely.

3. Add detector/describer support required by the paper.
   - Add the OpenCV contrib dependency needed for SURF.
   - Implement a detector-specific matcher/alignment service for the paper method so it does not disturb the current ORB path.
   - Reuse the existing route shape and homography-driven page-match result structure where possible.

4. Implement the paper's template construction pipeline.
   - Detect SURF keypoints/descriptors on the original template image.
   - Generate synthesized views from the single template image exactly from the paper's distortion families.
   - Geometric synthesis: simulate camera rotations and shifts using the paper's pinhole-camera-based homography approach, with 3 rotation angles sampled from `U(-45 deg, 45 deg)`.
   - Brightness synthesis: include monotone transformation of brightness by random function, global linear brightness transform, smooth brightness change along a random line, and darkening of one half-plane divided by a random line.
   - Blur/noise synthesis: include Gaussian blur, motion/defocus blur handling, normally distributed noise, and morphological closing as listed in the paper's experimental synthesis setup.
   - Use the known homography for each synthesized view to project original keypoints into the distorted image.
   - Compute keypoint stability exactly in the paper's form: count a match for keypoint `p` when some detected keypoint `q` in synthesized image `I_m` satisfies `||H_m(p) - q||_2 < epsilon`.
   - Use `epsilon = 5` as in the paper.
   - Synthesize `1000` images per template for stability estimation, matching the paper.
   - Rank template keypoints by stability `R(p)` and select the top `N` keypoints/descriptors to form the paper-method template artifact.

5. Persist paper-method artifacts per template.
   - Extend `backend/app/models/template_models.py` with lightweight metadata such as artifact path, detector type, version, and retained keypoint count.
   - Persist the varying-field mask metadata used to build the paper-method artifact.
   - Store keypoint and descriptor arrays in a binary sidecar artifact rather than inside `templates.json`.
   - Generate or rebuild the paper artifact when a template is created and when its image or artifact metadata changes.

6. Wire the new method into logical separation.
   - In `backend/app/api/extraction.py`, when the new method is selected, load the paper-method template artifact and evaluate each rendered PDF page with the SURF-based matcher.
   - Use brute-force descriptor matching for the paper method, matching the paper's evaluation pipeline choice instead of FLANN.
   - Keep the existing overall scoring/reporting shape so page diagnostics still stream to the UI.
   - Preserve clear error reporting when the paper artifact is missing or SURF is unavailable.

Paper-step checklist
- Single ideal exemplar image as starting point.
- Pre-marked varying/personal-data zones hidden from template keypoint selection.
- SURF keypoint detector and descriptor.
- Synthesized dataset built from the single image.
- Distortion families covered: geometric, brightness, blur, noise.
- Geometric transform generated through camera-model homographies.
- Stability score `R(p)` computed over synthesized views.
- `epsilon = 5`.
- `1000` synthesized images per template.
- Top-`N` stable keypoints retained for the template.
- Brute-force matching in the paper-based matcher.
- New method exposed alongside existing separation methods.
